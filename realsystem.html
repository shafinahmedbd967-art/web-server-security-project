Real System Check Dashboard (Linux Web Server Project)
1. Apache Server Status Check
Command
sudo systemctl status apache2
Expected Output
Active: active (running) Apache2 service is running normally
Use this to prove server is running.
2. Virtual Host Verification
URLs to open
http://localhost
http://site1.local
http://site2.local
Expected Output
Each URL loads a different webpage using Apache Virtual Hosting
Shows multiple websites running on same server.
3. HTTPS Security Check
Command (optional check)
sudo ss -tlnp | grep 443
Browser Test
https://localhost
Expected Output
Port 443 is listening Secure connection enabled (SSL active) Browser shows certificate warning (self-signed)
Confirms HTTPS is enabled.
4. Firewall Verification
Command
sudo ufw status
Expected Output
Status: active 80/tcp ALLOW 443/tcp ALLOW
Only required ports are open for security.
5. Live Access Log Monitoring
Command
sudo tail -f /var/log/apache2/access.log
Action
Open http://localhost and refresh browser
Expected Output
127.0.0.1 - - "GET / HTTP/1.1" 200 New log entries appear in real time
Demonstrates real-time monitoring.
6. Admin Panel Verification
URL
http://localhost/admin.html
Expected Output
Admin dashboard loads successfully Shows server status, monitoring, and system health
Basic management interface.
7. Security Hardening Proof
Configuration Lines
ServerTokens Prod ServerSignature Off Options -Indexes
Expected Output
Server does not expose version information Directory listing disabled Improved security posture
Prevents information leakage.
