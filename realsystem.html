Real System Check Dashboard (Linux Web Server Project - Viva Ready)
1. Apache Server Status Check
Command
sudo systemctl status apache2
Expected Output
● apache2.service - The Apache HTTP Server Active: active (running) Loaded: loaded (/lib/systemd/system/apache2.service)
✔ Server is running successfully and handling requests.
2. Virtual Hosting Verification
Step 1: Check enabled sites
ls /etc/apache2/sites-enabled/
Expected Output
site1.conf site2.conf
Step 2: Virtual Host mapping
apachectl -S
Expected Output
port 80 namevhost site1.local port 80 namevhost site2.local default server localhost
Step 3: Browser Test
http://site1.local http://site2.local
Expected Result
Site1 opens different webpage Site2 opens different webpage Same server hosting multiple websites
3. HTTPS Configuration Check
Enable SSL Module
sudo a2enmod ssl
Expected Output
Enabling module ssl To activate run: systemctl restart apache2
Check Port 443
sudo ss -tlnp | grep 443
Expected Output
LISTEN 0 128 0.0.0.0:443 apache2
Browser Test
https://localhost
Expected Result
⚠ Self-signed certificate warning appears ✔ Secure HTTPS connection established
4. Firewall Setup (UFW)
Command
sudo ufw status verbose
Expected Output
Status: active 80/tcp ALLOW 443/tcp ALLOW 22/tcp ALLOW
Proof Command
sudo ufw show added
Expected Result
Rules added: ALLOW IN 80 ALLOW IN 443 ALLOW IN 22
5. Live Access Log Monitoring
Command
sudo tail -f /var/log/apache2/access.log
Action
Open http://localhost and refresh browser
Expected Output
127.0.0.1 - - "GET / HTTP/1.1" 200 127.0.0.1 - - "GET /index.html" 200 Live logs updating in real time
✔ Confirms real-time request monitoring system.

Expected Output

Open Ports Check
ss -tlnp
Expected Output
:22 LISTEN :80 LISTEN :443 LISTEN
